Project 4: Visualizing YouTube Video Trends
Group Members: Britney Chen, Logan Evans, and David Freedman

The final product of our data visualization.

Introduction
For this project, we were instructed to gather data about interactions that took place on the internet. We then had to visualize this data in a way that could be drawn by an AxiDraw machine. We ultimately chose to focus on the data of different categories of videos that were on the trending page on Youtube.

We originally had the idea to visualize how humans have interacted with turtle migrations, but that yielded little information on the internet on how we have changed the migrations, or if we have changed their migration patterns. More specifically, We wanted to target how we as humans had affected the turtles through global warming but with global warming being a gradual process, finding both interesting and informative data proved to be difficult. We then found information that was more readily available to us: YouTube trending videos along with their categories.

Data Collection
Via a website in which users could submit datasets called Kraggle, a dataset of all the trending videos from Youtube from November 2017 to June 2018. This dataset has various spreadsheet for different countries and their corresponding trending videos. we used the US specific spreadsheet which can be downloaded from here. This particular spreadsheet is almost 41,000 cells long, each cell containing the following for each video:
  • a video id
  • the date it was trending
  • the title of the video
  • the channel title
  • an id for the category of the video
  • the time it was published
  • tags for the video
  • the number of views, likes, dislikes, and comments (each in its own column)
  • the link to the video
  • whether comments or ratings were disabled for the video
  • whether there is an error viewing the video or the video has been removed
  • the description for the video
In order to make the data more managable, we chose the date range of when the data started to the end of the month in which it started. This time range is November 14 - 30, 2017. We then focused on the category of each video in this range. The category for each cell was marked by a category id that corresponded to a type of category. For instance, the category id "1" corresponded to the category "Film and Animation". The creator of the dataset also had a file with all of the category ids along with the category title it represented so we could decifer what each id meant.

Designing the Data Visualization
There were 3,400 videos in our chosen date range and 14 different categories for these videos. Our next step was to figure out how to properly visualize the data in a creative but easy to understand manner. We came up with two different designs for our data.

One of our initial designs.

This is one of our initial visualization designs for our data. We wanted it to be colorful and simple, while still being portraying each trend. We also decided to have each arc / line be worth 10 videos, amounting to the total number of videos each trending category had. We wanted the art to have motion in it, which is why we chose to do arcs spiraling around a center point. We started with the largest amount of videos in a trend category and worked down to the category with the smallest amount of videos.

The second design for visualizing our data.

This was our second design, which we decided to use for this project. We wanted a way to represent not only the quantity of trending videos in each category, but also the relative proportions between them. Therefore, we came up with the idea to draw the data in rings around a center, each ring representing 10 videos in that category. Every ring is assigned a different height depending on the frequency of that video category, and so the end display is able to demonstrate the relative frequencies.

We chose this second design because it was simpler than the first design (we were worried the drawing would take too long) and fully completed in time for the drawing machine. We do think that the first design would be very interesting to see drawn by the machine and would be interested in doing that in the future.

Images of the AxiDraw producing our data visualization.

To produce the visualization using the AxiDraw, we had to convert the Illustrator file to an SVG file that contained the different color circle groups as different layers. We needed each color to be a different layer so that we could draw each portion individually, switching the pen colors between each layer. We used 8 different pen colors as that is what we had available to us. In the future, we would like to see our drawing with its originally chosen colors however with our lack of supplies, that was not possible for this stage. The file took a while to correctly format but once all the layers had been properly set and the dimensions were adjusted, we were able to start drawing. The process took about 2 hours including setup but resulted in our data visualization coming to life

A video of the AxiDraw drawing our design and the final product.
Final Reflection
Though we are pretty happy with our final product, there are a few things we would have done differently. We would be interested to see more data in our visualization along with using a date range in 2019 in order to see more up-to-date data. We also wish we had the proper pen colors when it was time to draw our design as it would have been nice to see the exact design created. We also would like to see the other design we made drawn by the machine as we think the end product would be quite interesting.
